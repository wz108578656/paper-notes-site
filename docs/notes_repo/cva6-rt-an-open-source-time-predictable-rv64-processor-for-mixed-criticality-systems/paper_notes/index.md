## 1. 摘要

**目的**  
为解决混合关键性系统中不同关键性任务间的资源竞争导致的时序干扰和延迟变异性问题，本文提出CVA6-RT，一个基于开源CVA6架构的实时微架构扩展，旨在提供有界的最坏情况延迟并降低任务执行时序的可变性。

**方法**  
- **可预测内存访问**：引入L1指令/数据缓存的动态可配置混合缓存/Scratchpad模式。每个缓存路可独立设置为传统缓存或软件管理的Scratchpad，Scratchpad区域从替换逻辑中移除，提供恒定延迟访问。  
- **可预测地址转换**：扩展伪LRU TLB替换逻辑，支持可配置的非重叠分区和条目锁定。通过位图选择活动分区，隔离不同任务的TLB替换，避免因驱逐产生的干扰。静态锁定条目可保证关键内存区域的恒定地址转换延迟。  
- **低延迟中断处理**：集成增强型RISC-V Core-Local Interrupt Controller (CLIC) ，支持细粒度优先级、嵌套和尾链。新增硬件辅助上下文保存机制，可在进入陷阱时自动将可配置寄存器组保存到预定义内存位置，消除软件堆栈开销。同时将中断检测逻辑从指令译码级移至提交级，并支持直接向低权限模式注入中断。

**结果**  
- **中断延迟**：CVA6-RT平均中断延迟为12个周期，较基线CVA6的140周期降低**10倍**。其中最大变异性从基线CVA6的170周期降至13周期。  
- **延迟分解**（见Table 1）：硬件上下文保存贡献从104-130周期降至0周期；中断注入延迟从6-30周期降至2-3周期，主要得益于检测逻辑后移和直接注入。  
- **可预测性**：结合TLB分区/锁定和Scratchpad模式，地址转换和内存访问延迟均有界且可分析。

**结论**  
CVA6-RT通过硬件级别的资源分区、锁定和加速中断处理，实现了与简化ARM Cortex-M系列微控制器相当的**12周期**中断延迟，并将变异性减少一数量级。其设计使得时间关键任务能够获得有界、可预测的响应时间，适用于混合关键性系统中的软实时工作负载。

## 2. 背景知识与核心贡献

**研究背景**  
混合关键性系统需在单个处理器上同时运行性能导向任务和具有最差情况延迟界限的控制任务。当前汽车与机器人领域趋向于将更多处理集中到单个SoC（如区域控制器），导致资源争用加剧，任务间干扰和延迟变异性显著。软件方法（如缓存/TLB着色）延迟开销高；硬件方法（将SoC分为32位/64位关键域）增加成本、降低灵活性，且可能造成资源闲置。

**研究动机**  
业界已有ARM Cortex-R等具备原生实时硬件支持的64位处理器，并集成于汽车区域控制器中。RISC-V生态中缺乏具有类似实时能力的开源应用级处理器。因此，亟需为RISC-V架构提供原生硬件级时间可预测性机制，以支撑混合关键性工作负载。

**核心贡献**  
本文提出 **CVA6-RT**，在开源CVA6 RISC-V处理器基础上进行的实时微架构扩展，具体包括：

- **可预测地址翻译**：扩展TLB替换逻辑，支持软件配置的**TLB分区**（使用位图选择活动分区）和**静态锁定**部分条目，消除任务间TLB置换干扰，保证关键内存区域地址翻译延迟恒定。
- **确定性内存访问**：在L1指令缓存和数据缓存中引入可运行时配置的**混合缓存/Scratchpad模式**，每个缓存路可独立配置为常规缓存或软件管理Scratchpad。Scratchpad区域从替换逻辑中移除，提供恒定延迟的本地内存访问。
- **低延迟中断处理**：集成增强的**RISC-V CLIC**（支持细粒度优先级、抢占、尾链），并增加**硬件辅助寄存器堆栈**。中断入口时自动保存可配置的寄存器组，消除软件上下文保存开销。中断检测逻辑移至提交阶段，使中断同步延迟大幅降低。

**性能结果**：在中断延迟测试中，CVA6-RT达到**12个周期**的平均中断延迟，相比基线CVA6（140周期）降低**10倍**；延迟变异性也从50周期降至仅1周期。

## 3. 核心技术和实现细节

### 0. 技术架构概览

**整体技术架构**

CVA6-RT 基于开源的 **64位 RISC-V 应用级处理器 CVA6**（支持虚拟内存、私有 L1 指令/数据缓存、完整操作系统栈）进行微架构增强，旨在为混合关键性系统提供时间可预测性。其架构核心包含三个主要硬件扩展模块，如图 `images/93d460cfed2e93a654ac2ad5bd389344eca26ec56a9c3e9e413bc2df6954e7df.jpg` 所示。

- **可预测内存访问延迟**：在 L1 指令和数据缓存中引入混合 **缓存/暂存器 (SPM)** 模式。
  - 每个缓存路可动态配置为传统缓存路或软件管理的**暂存器**。
  - 配置为 SPM 的路会从替换逻辑中移除，并清除标签和有效位，避免意外命中。
  - 地址解码逻辑将指定物理地址区域映射到 SPM，并将其集成到 SoC 地址空间中。
  - SPM 区域组织为连续缓存路以简化硬件映射，提供恒定的单周期访问延迟，不受缓存替换效应影响，适用于中断处理程序和时间关键代码。

- **可预测地址翻译**：扩展原生的 **伪LRU TLB替换逻辑**，支持 **TLB条目分区**和 **静态锁定**。
  - **分区**：通过软件配置位图将 TLB 条目划分为互不重叠的集合，任务被分配独立分区后可隔离替换干扰，防止因共享核引起的TLB驱逐冲突。
  - **锁定**：可静态锁定指定数量的TLB条目，使其从替换机制中移出且不可被驱逐，保证关键内存区域的地址翻译延迟恒定。

- **快速中断处理**：集成增强型 **RISC-V 核本地中断控制器 (CLIC)**，并扩展陷阱处理逻辑以支持**硬件辅助上下文保存**。
  - CLIC 提供细粒度中断优先级和抢占，支持低延迟向量化、嵌套和尾部链，并扩展了**虚拟化支持**，在虚拟化系统中仍保持低且可预测的延迟。
  - 硬件上下文保存：在中断入口时，核心可自动将可配置的子集寄存器溢出到预定义内存位置，消除软件保存的延迟和抖动，将陷阱进入开销降至接近流水线刷新延迟。

这三个模块协同工作：TLB 分区与锁定保障地址翻译确定性，缓存/SPM 混合模式保障内存访问确定性，增强的 CLIC 与硬件上下文保存保障中断响应确定性，使 CVA6-RT 在保持 RISC-V 开放生态的同时，实现与 ARM Cortex-M 相当的中断延迟（12 个周期，较基线 CVA6 降低 10 倍）和极低的时序可变性。

### 1. TLB Partitioning and Locking

**实现原理**

- **基础架构**：CVA6-RT在原有CVA6的伪LRU（pseudo-LRU）TLB替换逻辑基础上进行扩展。伪LRU使用二叉树结构记录最近使用情况，以近似LRU策略决定替换候选。
- **分区约束**：通过在伪LRU树中引入额外的**约束逻辑**，将TLB条目集合划分为多个**互不重叠的固定分区**。每个分区对应一组连续的TLB条目（entry index范围）。
- **锁定机制**：独立于分区，提供**静态锁定**（locking）机制。锁定条目从伪LRU替换候选集中移除，其有效位被标记为锁定状态，后续替换操作跳过这些条目。

**算法流程**

1. **初始化配置**：特权软件（如操作系统内核）通过**内存映射控制寄存器（CSR）或专用接口**写入一个**分区位图（bitmap）**。位图中每个比特对应一个分区标识符，表示该分区当前是否被激活（分配给某个任务或关键性域）。
2. **分区分配**：每个任务或关键性上下文被赋予一个或多个分区ID。当任务运行时，TLB替换逻辑仅在其对应激活分区内的条目中进行替换。若任务使用多个分区，则在这些分区内共同替换。
3. **替换决策**：当TLB缺失发生时，硬件先在当前任务激活分区内的条目中，根据伪LRU树的状态选择一个受害者条目。**非激活分区中的条目不会被选为牺牲项**，从而实现了任务间替换隔离。
4. **锁定配置**：通过另一组CSR指定要锁定的TLB条目索引。锁定后，这些条目不再参与伪LRU树更新和替换决策，且不会被任何后续的TLB缺失所覆盖。
5. **动态重配置**：分区和锁定状态可在运行时通过特权软件修改，但需保证修改期间无相关TLB缺失（通常通过同步屏障（fence）指令实现）。

**参数设置**

- **分区数量**：可配置，由硬件实现寄存器宽度（比如32个分区，对应32比特位图）。
- **每个分区的大小**：由初始TLB总条目数除以分区数，或者支持非均匀分区（通过基址/长度寄存器）。文档中描述为“non-overlapping sets”，通常采用等分或由软件指定边界。
- **锁定条目数**：可配置，通过一个计数寄存器限制最多锁定的条目数量，防止过度锁定导致TLB可用空间不足。
- **激活位图**：一个与分区数等宽的比特掩码，每个比特1表示对应分区被激活。特权软件按任务写入该值。

**输入输出关系**

- **输入**：
  - **特权软件配置**：分区位图（bitmap）、锁定条目列表、任务切换时更新的激活位图。
  - **硬件事件**：TLB缺失信号、当前任务标识（用于选择激活分区集）。
- **输出**：
  - **TLB替换结果**：被替换的条目索引（仅来自激活分区中的候选）。
  - **地址翻译延迟**：对于锁定条目，保证恒定单周期访问；对于分区内条目，延迟可控但可能因伪LRU状态略有抖动。整体上最大化减少跨任务干扰。
- **在整体中的作用**：作为CVA6-RT**可预测地址翻译**的核心机制，消除因不同任务争用TLB导致的替换抖动，为关键任务提供确定性的虚拟地址到物理地址转换延迟，从而支持混合关键性系统中实时任务的时序可分析性。

**性能影响**

- 相比于无分区/锁定的基线CVA6，CVA6-RT的TLB命中延迟不变（1周期），但缺失延迟可能因分区约束微增（额外比较逻辑），但整体确定性显著提升。
- 锁定条目保证100%命中，用于中断向量表、实时任务栈等关键代码区域。

### 2. Hybrid Cache/Scratchpad Mode in L1 Caches

**核心观点**：Hybrid Cache/Scratchpad Mode 通过在 L1 缓存中引入软件可配置的暂存器路，将部分缓存资源从动态替换中隔离，从而为时间关键任务提供**恒定延迟**的内存访问路径。

**实现原理**

- **缓存路动态配置**：每个 L1 缓存（指令和数据）的物理路被设计为可独立配置的模式位。通过特权软件（如操作系统）写入配置寄存器，可将任意路标记为 **常规缓存路** 或 **暂存器路**。
- **替换逻辑隔离**：所有被标记为暂存器的路从缓存替换算法（原为伪 LRU）中**彻底移除**。这意味着它们不再参与命中/缺失判断，也不会被未命中事件驱逐。
- **标签与有效位清零**：为阻止意外缓存命中（因残留的脏数据或旧标签），暂存器路的标签数组和有效位被**硬件强制清除**，确保任何来自CPU的地址请求都不会将其视为缓存命中。
- **地址解码映射**：在缓存控制器中增加一个**地址解码逻辑单元**，该单元将一段或几段连续的物理地址区域（由基地址和大小寄存器定义）直接映射到这些暂存器路。当CPU对映射地址发起访问时，地址解码逻辑会跳过常规缓存标签比较，直接将请求路由到对应的暂存器路存储体。

**算法流程**

- **配置阶段**：操作系统将一段连续的物理内存区域（例如0x1000~0x1FFF）设置为暂存器模式，并指定使用L1数据缓存的第0路和第1路作为其物理存储。硬件自动清除这两路的标签，并更新地址解码映射表。
- **访问阶段**：当CPU执行加载/存储指令，目标物理地址命中映射区域时，控制器**不经过标签比较**，直接计算暂存器路内的索引偏移，在**固定一个时钟周期**（如2个周期）内完成数据读写。若目标地址不在映射区域，则照常进行常规缓存查找（命中/缺失）。
- **动态切换**：软件可在运行时重新配置模式位，将暂存器路还原为常规缓存路（需先清空或回写暂存器数据），或增加/减少暂存器路数。

**参数设置**

| 参数 | 说明 | 默认值（示例） |
|------|------|----------------|
| 可配置的路数 | L1缓存总路数（例如4路或8路）中可动态分配给暂存器的最大数量。 | 4路中的2路 |
| 映射地址区域 | 一组连续的物理地址范围，由基地址和大小寄存器定义。 | 由软件指定，例如 `0x8000_0000` ~ `0x8000_FFFF` |
| 路组织连续性 | 暂存器路必须为**连续的缓存路**（例如路0和路1，不可跳选路0和路2），以简化地址解码硬件。 | 硬件强制约束，软件需遵守 |

**输入输出关系**

- **输入**：CPU发出的**物理地址**（包含在加载/存储指令中）以及读/写使能信号。
- **输出**：若物理地址落在暂存器映射区域，则输出**恒定延迟**（例如2个周期）的数据或写入确认；若不在映射区域，则输出常规缓存访问的延迟（可能因替换、缺失而波动）。
- **整体作用**：该机制为时间关键型代码和数据（如中断向量、实时任务栈、调度器数据）提供了一个**确定性**的本地内存空间，消除了因缓存替换带来的时序抖动。结合TLB分区和低延迟中断处理，使CVA6-RT能够为**混合关键性系统**中的软实时任务提供可分析的最差情况执行时间（WCET）。

### 3. Low-Latency Interrupt Handling with Enhanced CLIC and Hardware-Assisted Context Save

**核心观点**  
CVA6-RT通过集成增强型RISC-V **Core-Local Interrupt Controller (CLIC)** 与硬件辅助的**上下文保存机制**，将中断延迟从基线CVA6的120–170周期压缩至**12–13周期**，且抖动降至1周期以内。该设计消除了软件保存开销，并优化了中断检测与注入路径，为混合关键性系统提供确定性的低延迟中断响应。

---

**增强CLIC的实现原理**  
- CLIC提供**细粒度中断优先级**（支持最多256级）和硬件优先级抢占，使高优先级中断可立即打断低优先级处理，无需软件仲裁。  
- 支持**直接向量化**：中断发生后，CLIC直接提供目标中断服务例程（ISR）的PC，省去软件查表跳转。  
- 支持**中断嵌套**和**尾链（tail-chaining）**：在返回时若等待另一中断，可跳过多余上下文恢复/保存，减少流水线停顿。  
- 关键微架构改动：将中断检测逻辑从**指令解码阶段**移至**提交（commit）阶段**，使中断采样更早且与流水线状态同步，消除因解码阶段空泡导致的额外延迟（原CVA6中该部分贡献6–30周期可变延迟）。  
- 扩展支持**虚拟化中断注入**：允许hypervisor直接向低特权级（如Guest OS）注入中断，在虚拟化场景下保持低延迟（原需两级trap转换）。

**硬件辅助上下文保存的实现原理**  
- 通过扩展trap处理逻辑，增加**硬件状态机**，在中断入口自动将寄存器保存到预定义内存区域（软件可配置寄存器子集，如通用寄存器x1–x31的任意组合）。  
- 保存操作与**流水线冲刷**并行执行：在flush指令阶段后，硬件状态机占用存储端口将寄存器值写入栈空间，不额外消耗流水线周期。  
- **参数设置**：软件通过CSR（控制状态寄存器）配置保存哪些寄存器以及存储基址。设计允许动态调整保存集，平衡实时性与功能需求。  
- 完全消除软件保存所需的**104–130周期**（如表1所示），且因为硬件操作固定，延迟变异性降为0。

**输入输出关系及在整体架构中的作用**  
- **输入**：外部中断信号（IRQ）抵达CLIC；当前CPU特权模式与中断优先级状态；硬件上下文保存配置CSR。  
- **输出**：  
  - 中断向量化目标地址（PC）送入取指单元；  
  - 硬件状态机产生的存储请求（将寄存器写入内存）；  
  - 修改后的控制状态（如特权级提升、中断屏蔽位更新）。  
- **作用**：该机制是CVA6-RT实时性的核心支柱——它确保中断响应时间具有**有界且可分析**的低延迟，为混合关键性系统中的时限任务（如控制循环、安全监测）提供硬件保障。结合TLB分区和缓存/Scratchpad模式，CVA6-RT可将整个关键软件路径（从中断触发到任务切换）的时序不确定性降至最低。

**总结**  
该技术通过**CLIC的微架构优化**（检测前置、优先级硬件管理、向量化）和**硬件上下文保存**的并行化执行，彻底消除了软件中断入口的开销。它使CVA6-RT在中断延迟上媲美简单Cortex-M处理器，同时保留了64位应用处理器的虚拟内存与OS支持能力，成为混合关键性系统中时间关键任务与通用任务共存的基石。

## 4. 实验方法与实验结果

**实验设置**
- 论文以**中断延迟**作为代表性用例评估CVA6-RT的实时性能。
- 实验平台为RTL级仿真，对比基线CVA6与CVA6-RT在相同工作负载下的中断响应时间。
- 中断延迟分解为四个组成部分：IRQ propagation（中断信号传播）、IRQ injection（中断注入到流水线）、pipeline flush（流水线冲刷）、context save（上下文保存）。每个部分均测量最小和最大延迟（cycle数）。

**结果数据**
- 平均中断延迟：CVA6-RT为**12 cycles**，基线CVA6为**140 cycles**，实现**10×降低**。
- 具体数值如表1所示：
  - **IRQ injection**：CVA6 (min 6, max 30) → CVA6-RT (min 2, max 3)。降低得益于中断检测逻辑从指令译码级移到提交级，以及支持向低特权模式的直接中断注入。
  - **Context save**：CVA6 (min 104, max 130) → CVA6-RT (min 0, max 0)。**硬件辅助寄存器堆栈消除软件开销**，是最大改善来源。
  - IRQ propagation和pipeline flush在两者中保持不变，分别为3 cycles和7 cycles。
  - 总延迟范围：CVA6 (120–170 cycles), CVA6-RT (12–13 cycles)。**CVA6-RT的延迟抖动量也大幅降低**（仅1 cycle抖动）。

**消融实验分析**
- 表1本质上是一种**组成部分分解的消融**：通过单独评估每个硬件改进对中断延迟的贡献，量化各机制的效果。
- **关键发现**：
  - **硬件上下文保存**是最大收益来源，将保存寄存器的时间从>100 cycles降为0，同时消除所有相关抖动。
  - **优化中断注入**将延迟从6–30 cycles稳定至2–3 cycles，主要受益于流水线中更早的中断识别与特权模式切换改进。
  - **TLB分区与缓存scratchpad模式**虽然在此特定实验中未直接体现，但论文指出这些机制为预测性内存访问和地址转换提供基础，与中断低延迟共同保障**混合关键性系统的可分析性**。
- 其他组成部分（信号传播、流水线冲刷）未作修改，说明CVA6-RT在不改变核心流水线深度前提下，通过外围协处理器（CLIC）和专用硬件堆栈实现了实时性提升。

**结论与意义**
- CVA6-RT以12 cycle的中断延迟达到与简单ARM Cortex-M微控制器相当的水平，而基线CVA6作为应用级核心需140 cycles。这一成绩验证了所提出的实时微架构扩展在**开放RISC-V生态**中填补了缺乏应用级实时处理器的空白。
- 消融结果凸显**硬件辅助机制**（特别是寄存器堆栈）对于消除软件处理带来的不可预测延迟至关重要，同时**细粒度中断控制**（CLIC）与**流水线优化**协同作用显著降低了延迟均值和方差。
